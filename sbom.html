The problem Atlas SBOM solves
"CRA requires software manufacturers to maintain a complete SBOM and disclose vulnerabilities — but most teams don't even know what format their tools produce, let alone whether it will hold up to a regulator."
The EU Cyber Resilience Act deadline is December 2027 — and the SBOM requirements are non-trivial. Meanwhile, the tooling landscape is fragmented: GitHub exports SPDX, Dependency-Track dropped SPDX, and teams spend more time on format conversion than on actual compliance work. Atlas SBOM was built to eliminate that overhead and give you a clear path to CRA readiness.

Format support Native ingestion, no conversion required
CycloneDX

The OWASP standard optimised for security automation. Supports all CycloneDX versions including v1.6 and v1.7. Ingests XML and JSON variants directly.

Natively supported
SPDX

GitHub's native SBOM export format. Also common in compliance and legal workflows. Atlas SBOM ingests SPDX 2.x and SPDX 3.0 without any intermediate conversion step.

Natively supported
GitHub Native

GitHub's dependency graph SBOM export produces SPDX directly. Atlas SBOM ingests this output as-is — no pipeline modifications, no format conversion, no extra tooling required.
